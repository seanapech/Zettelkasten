## Citations/Sources

####  Introduction, Paragraph 1, Line 1

	Quote: "Public-key cyptography, or asymmetric cryptography, is the field of cryptographic systems that use pairs of related keys."

Essentially, public-key cryptography is the process of generating private and public keys that must be used in conjunction for authentication/verification. This kind of cryptography is also known as one-way functions, where a public key can be generated via a mathematical algorithm which can only be decrypted by its private key pair, and cannot be put through the same algorithm to reproduce the private key.

####  Introduction, Paragraph 1, Line 4

	Quote: "Security of public-key cryptography depends on keeping the private key secret; the public key can be openly distributed without compromising security."

Public-key cryptography takes the burden of [symmetric cryptography] off of the user to always protect their key and always use some kind of secure exchange tunnel/platform, and shift the burden to the carrier of the private key. This means you can have any number of public keys that are compatible with a single private key, but never compromise the security of this authentication method if you share any of those public keys.
